Time and Revision Search

route: GET "/api/time"
return: JSON
format: /api/time?[titles={titles1|titles2|...}][pageids={pageid1|pageid2|...}]

Filter Parameters

By titles (Optional)
/api/time?titles=Zero_no_Tsukaima|Kino_no_Tabi

Either parameter may be present. Multiple values must be separated by a pipe character ("|"). The values must be a valid interwiki page link.

By pageids (Optional)
/api/time?pageids=1912

Either parameter may be present. Multiple values must be separated by a pipe character ("|"). The values must be a valid wiki pageid.


JSON properties

A list of JSON is returned as to preserve the order. Titles queries go before page queries.

title

(STRING) The title of the page.

pageid

(NUMBER) The pageid of the page.

lastrevisedid

(NUMBER) The unique id of the last revision of the page.

lastreviseddate

(STRING) The date and time of the last revision of the page.

Example response using

/api/time?titles=Zero_no_Tsukaima|Kino_no_Tabi&pageids=1912

Latest Update Search

route: "/api/time"
return: JSON
format: /api/time?updates={number_of_newest_pages}[&from={starting_datetime}][&until={ending_datetime}]

Filter Parameters

By number of updates
/api/time?updates=20

This must be a number. Extra characters are removed.

By starting date and time (Optional)
/api/time?updates=20&from=2015-04-22T14:57:31Z

The date and time must be in the form of UTC+0, YYYY-MM-DDTHH:MM:SSZ. This datetime marks the end of the search, as the server searches from the current datetime.

By ending date and time (Optional)
/api/time?updates=20&from=2015-04-22T14:57:31Z&until=2015-04-22T20:58:55Z

This must be used with the from parameter. If not, it will be ignored. The date and time must be in the form of UTC+0, YYYY-MM-DDTHH:MM:SSZ


JSON properties

A list of JSON is returned as to preserve the order.

title

(STRING) The title of the page.

pageid

(NUMBER) The pageid of the page.

revid

(NUMBER) The id of the page that is created.

timestamp

(STRING) The date and time of the creation of the page.

Example response using

/api/time?updates=5&from=2015-09-29T00:32:56Z&until=2015-09-30T08:42:23Z
